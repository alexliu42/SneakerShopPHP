Documentation

The final project is an online sneakers web app that displaying the available trending shoes for clients to make reservation. There will be a web form asking for the information from clients during reservation. The data will be then stored and updated in our database.

The structure and detail of the web app is as below:

Home page:

  • A basic home page that displays the name and contact information of the online sneaker store using two columns layout.
  • A simple contact form that directs to the user email service.

Album page:

  • The album page displays all the available shoes in stock with their pictures using grid layout.

Reservation page:

  • The page contains a reservation form to ask clients to fill in their contact information to further contact by phone or email.
  • The contact information, Name, Email and Phone are set to be requiring fields in the valdiation
  • There is a selection dropdown displaying the available shoes fetching from database for user to select.
  • There will be dynamic values displaying for the size of the shoes after selection
  • There is an add shoes button to dynamically generate more sections to add shoes
  • The maximum numbers of shoes to reserve is set to be 3 shoes, an indicating message will show up if the user tries to add more than 3 shoes

Documentation page:

  • The documentation page gives the detail of the web app.

SQL:

  • There are two tables in database. One table stores the information of clients contact, and the other table stores the information of shoes.
  • The table of contact will be spontaneously updated once the clients submit the reservation form.

The reservation web app is implemented using Html, CSS, JavaScript, PHP, JQuery and MYSQL to perform the client server-side communication
